I built an easy multi-player, multi-screen computer science game using Node.js and the Socket.IO library.

Node.js is a server-side JavaScript framework. It means that the very code behind the very popular Gmail can also power servers too! The secret sauce is in the asynchronous design, which allows for it to handle millions of concurrent connections at once and I don’t have to wait for a slow file input/output or databases for processing. To allow for two different players to play the game at the same time, I used Socket.io, a realtime library for Node.js. The source code is here

As you'd be able to see in the source code, another fun thing I learned was how to use JavaScript class declarations, which provide a much simpler syntax to make objects and deal with inheritance. I learned this thanks to my code mentor, and it was especially helpful because I almost went down the less efficient path and wrote a function to listen for each type of event. For example, I wrote Game and Player classes so they could inherit from the EventEmitter.prototype (from socket.io), allowing us to use instances of these 'classes' to emit and listen to events.
